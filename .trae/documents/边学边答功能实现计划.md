# 边学边答功能实现计划（通用课程导入版）

> 本文档为技术规范文档，涵盖导入逻辑的详细技术说明、分步骤实现指南、关键技术点解析、潜在问题处理方案以及完整的使用操作说明。

## 一、Summary（功能概述）

在 Android 原生端（Kotlin + Compose）实现"边学边答"功能模式。**不绑定任何固定课程，不生成系统内置格式数据**，仅实现通用的「课程包」可导入功能：用户可导入任意"教程 + 配套题目"组成的课程包 JSON 文件，系统自动解析为「知识点章节 + 关联题库」，实现学完每个知识点后立即答题巩固。功能包括：课程包导入与解析、知识点讲解展示、配套答题、学习进度跟踪与效果检验。

## 二、设计理念：教程 + 教程题目 关联格式

一个「课程包」是一个 JSON 文件，包含两部分并相互关联：

- **教程（course）**：结构化的知识点章节，每章含讲解、代码示例
- **教程题目（questions）**：配套练习题，每题通过 `knowledgePoints` 字段标注所属章节

两者通过「章节标识」关联：教程章节的 `id`（如 `ch1_3`）与题目的 `knowledgePoints`（如 `["ch1_3"]`）一一对应。导入后即可实现"学完某教程章节 → 答对应教程题目"。

## 三、现有导入逻辑架构技术分析

> 本节是对项目现有导入体系的代码级技术分析，为后续扩展"课程包"格式提供架构依据。

### 3.1 整体架构概览

导入系统采用三层架构，数据流清晰单向：

```
UI 层 (ImportScreen.kt)
    ↓ 文本/文件 Uri
解析分发层 (QuizRepository.parseImportJsonPreview / QuizImportParser)
    ↓ ImportResult (题目列表 + 诊断)
数据持久层 (QuizRepository.importBank / appendQuestionsToBank → persist)
```

关键设计特点：**JSON 与纯文本走两条独立解析路径**，在 `ImportScreen.startParse()` 中先尝试 JSON，失败则回退到文本解析器。

### 3.2 UI 层：ImportScreen.kt 文件选择与导入流程

**文件**：`apps/android/app/src/native/java/com/yiqiu/shirohaquiz/ui/screens/ImportScreen.kt`

#### 文件选择器实现

使用 Android Storage Access Framework (SAF)，通过 `ActivityResultContracts.OpenDocument()` 注册 launcher：

```kotlin
// 主文件选择器（ImportScreen.kt:255-302）
val filePicker = rememberLauncherForActivityResult(ActivityResultContracts.OpenDocument()) { uri ->
    if (uri == null || isImportBusy) return@...
    selectedFileName = queryFileName(context, uri)
    // 文件大小校验 → 读取字节 → 解码
}
```

启动方式（`ImportScreen.kt:660`）—— MIME 类型使用 `arrayOf("*/*")`，接受所有类型，由解码器根据内容判断格式：

```kotlin

```

**关键技术点**：无需任何运行时权限声明。`OpenDocument()` 返回的是临时 `content://` Uri，SAF 自动授予细粒度读取权限。

#### 文件读取链

1. `ContentResolver.query(uri)` 查询文件名（`_display_name` 列）
2. `OpenableColumns.SIZE` 校验文件大小（分警告/阻断两档）
3. `openInputStream(uri)?.readBytes()` 读取全部字节
4. 交给 `QuestionImportAssetExtractor.decodeDetailed()` 解码（支持 docx/txt/json/xlsx/csv + 图片提取）

#### 解析触发与分发（核心：startParse）

`startParse()`（`ImportScreen.kt:348-420`）是解析调度中枢，**分发优先级链**：

1. **JSON 单题库** → 直接构造 `ImportResult`
2. **JSON 多题库** → 抛异常，引导去"数据管理"
3. **双文件文本** → `QuizImportParser.parseDualText()`
4. **标准文本** → `QuizImportParser.parseStandardText()`

```kotlin
fun startParse() {
    importScope.launch {
        val result = runCatching {
            withContext(Dispatchers.Default) {
                // 第一步：优先尝试 JSON 预解析
                val jsonPreview = QuizRepository.parseImportJsonPreview(rawSnapshot)
                // 第二步：多题库 JSON 拦截
                if (jsonPreview != null && jsonPreview.banks.size > 1) {
                    throw IllegalArgumentException("...")
                }
                // 第三步：单题库 JSON 直接构造结果
                val jsonBank = jsonPreview?.banks?.singleOrNull()
                val parsedResult = if (jsonBank != null) { ... }
                else if (dualSnapshot) { QuizImportParser.parseDualText(...) }
                else { QuizImportParser.parseStandardText(...) }
            }
        }
    }
}
```

### 3.3 解析分发层：QuizRepository JSON 格式识别

**文件**：`apps/android/app/src/native/java/com/yiqiu/shirohaquiz/state/QuizRepository.kt`

#### JSON 格式识别核心逻辑

`parseStandaloneJsonPreview()`（`QuizRepository.kt:2373-2422`）是格式分发的关键：

```kotlin
private fun parseStandaloneJsonPreview(rawText: String): JsonImportPreview? {
    val text = rawText.trim()
    return if (text.startsWith("[")) {
        // 分支 A：顶层数组
        val array = JSONArray(text)
        if (looksLikeBankArray(array)) { /* 多题库数组 */ }
        else { /* 题目数组 */ }
    } else {
        // 分支 B：顶层对象
        val root = JSONObject(text)
        val standaloneBank = parseStandaloneBankJsonObject(root)
        when {
            standaloneBank != null -> { /* 单题库 */ }
            root.optJSONArray("banks") != null -> { /* 多题库备份 */ }
            else -> null  // 回退到文本解析
        }
    }
}
```

**格式识别决策树**：

| 输入特征 | 识别 kind | 处理方式 |
|---|---|---|
| `[` 开头 + 首元素含 `questions` | `multi_bank_array` | 多题库，导入页拦截 |
| `[` 开头 + 首元素无 `questions` | `question_array` | 单题库（题目数组） |
| `{` 含 `questions` 字段 | `single_bank` | 单题库 |
| `{` 含 `banks` + Web 标志 | `web_backup` | 多题库 + 跨端状态 |
| `{` 含 `banks` 无 Web 标志 | `multi_bank_backup` | 多题库 |
| 其他 | `null` | 回退到文本解析 |

> **扩展点**：课程包格式（含 `course` 节点）的识别，只需在分支 B 的 `when` 表达式中新增一个分支。

#### 题目解析的高度兼容性

`parseQuestion()`（`QuizRepository.kt:4489-4595`）对所有字段都做了多别名兼容：
- **选项**：支持 `[{key,text}]` 数组、`["选项A"]` 字符串数组、`{"A":"文本"}` 对象三种格式
- **答案**：支持 `answer`/`answers`/`correctAnswer`/`correctAnswers` 四种字段名，支持数组和分隔符字符串
- **题型**：`parseQuestionType()` 支持大量别名（single/single_choice/radio 等）
- **元数据**：已解析 `knowledgePoints`、`difficulty`、`category`、`tags` 等字段

> **关键结论**：题目解析层无需任何改动，课程包的 `questions` 部分可直接复用全部解析能力。

### 3.5 完整数据流总结

```
用户操作
  ├─ 选择文件 → filePicker.launch(*/*) → Uri
  │     → ContentResolver.openInputStream → bytes
  │     → QuestionImportAssetExtractor.decodeDetailed → DecodeResult(text)
  │     → rawText = content.text
  ├─ 粘贴文本 → OutlinedTextField → rawText
  └─ 点击"开始解析" → startParse()
        → Dispatchers.Default 异步
        ├─ JSON 路径: parseImportJsonPreview → parseStandaloneJsonPreview
        │     → 识别 kind → JsonImportPreview(banks)
        │     → 单题库: 直接构造 ImportResult(questions)
        └─ 文本路径: QuizImportParser.parseStandardText / parseDualText
              → 多候选竞争 → ImportResult(questions, warnings)
        ↓
  applyParsedResult() → editableQuestions
  ↓ 预览 UI
  ↓ 确认写入
  ├─ NEW_BANK → importBank → banks += → persist()
  └─ APPEND_TO_BANK → appendQuestionsToBank → persist()
```

## 四、课程包格式定义（核心规范）

### 4.1 课程包 JSON 结构

导入的「课程包」JSON 格式如下（兼容现有题目解析器，教程与题目共存于一个文件）：

```json
{
  "course": {
    "courseId": "kotlin-basic",
    "courseName": "Kotlin 程序设计基础（一）基础语法",
    "description": "Kotlin 入门基础语法，边学边练",
    "sections": [
      {
        "id": "ch1_3",
        "chapterTitle": "第1章 走进新语言",
        "sectionTitle": "1.3 程序代码基本结构",
        "order": 2,
        "summary": "一句话摘要",
        "content": "详细讲解（纯文本，\\n 分段）",
        "codeExample": "示例代码",
        "codeExplanation": "代码要点",
        "difficulty": "easy",
        "prerequisites": ["ch0"],
        "questionTag": "ch1_3"
      }
    ]
  },
  "bankName": "Kotlin基础语法题目",
  "questions": [
    {
      "type": "single",
      "question": "Kotlin 程序的入口函数是？",
      "options": [
        {"key": "A", "text": "fun main()"},
        {"key": "B", "text": "public static void main"}
      ],
      "answer": ["A"],
      "analysis": "Kotlin 程序入口是 fun main()",
      "knowledgePoints": ["ch1_3"],
      "category": "1.3 程序代码基本结构",
      "difficulty": "easy"
    }
  ]
}
```

### 4.2 关联规则

题目 `knowledgePoints` 中的值（如 `"ch1_3"`）与教程章节 `id`/`questionTag` 匹配，即构成"教程章节 → 教程题目"的对应关系。

**容错**：若题目未标注 `knowledgePoints`，仍可作为普通题库使用，只是不会出现在任何章节的练习中。

## 六、关键技术点解析

### 6.1 课程包格式识别（复用现有架构）

现有 `parseStandaloneJsonPreview()` 已具备完善的 JSON 格式识别能力。课程包格式的识别只需在顶层对象的 `when` 分支中新增一个判断：

```kotlin
// 在 parseStandaloneJsonPreview 的 else 分支中新增
val courseObject = root.optJSONObject("course")
when {
    standaloneBank != null -> { /* 现有：单题库 */ }
    courseObject != null -> { /* 新增：课程包格式 */ }
    banksArray != null -> { /* 现有：多题库 */ }
    else -> null
}
```

但课程包导入走独立入口（StudyScreen），不经过 ImportScreen 的 `startParse()`，因此**无需修改现有导入页的格式识别逻辑**，避免影响现有题库导入功能。

### 6.2 题目解析零改动复用

`parseQuestion()` 已支持 `knowledgePoints` 字段的解析（`parseStringArray`），课程包的 `questions` 部分可直接通过 `importBank()` 导入，无需任何改动。

### 6.3 持久化扩展遵循现有范式

`knowledgeCourses` 使用「嵌套对象列表」范式（仿 `studyRecords`），`studyProgress` 使用「简单 Map」范式（仿 `practiceSequentialProgress`），与现有代码风格完全一致。

### 6.4 文件选择器零成本复用

课程导入的文件选择直接复用 `ActivityResultContracts.OpenDocument()`，无需运行时权限，无需修改 Manifest。

## 七、潜在问题处理方案

### 7.1 课程包格式校验失败
**问题**：用户导入的 JSON 不含 `course` 节点或格式不合法。
**处理**：`importCoursePackage` 内部用 `runCatching` + `JSONObject.optJSONObject("course")` 校验，失败时返回明确的错误提示（如"未检测到课程包格式，请确认 JSON 包含 course 节点"）。

### 7.2 重复导入同一课程
**问题**：用户多次导入同一个课程包。
**处理**：`importCoursePackage` 检查 `courseId` 是否已存在，若存在则提示"该课程已导入，是否重新导入？"，确认后删除旧课程与关联题库再重新导入。

### 7.3 章节无关联题目
**问题**：某个章节的 `questionTag` 没有匹配到任何题目的 `knowledgePoints`。
**处理**：章节仍可正常学习知识点，但"开始练习"按钮显示"本章暂无题目"并禁用。不阻断学习流程。

### 7.4 课程删除的数据一致性
**问题**：删除课程时需同步删除关联题库和进度。
**处理**：`deleteCourse()` 依次清理：`studyProgress` 中该课程的所有进度 → `knowledgeCourses` 中的课程 → 关联的 `QuizBank`（调用现有 `deleteBank()`）。错题本中引用该题库的条目由现有 `deleteBank` 自动清理。

### 7.5 大文件导入性能
**问题**：课程包文件较大时解析可能卡顿。
**处理**：解析逻辑放在 `Dispatchers.Default` 异步执行（参考 `startParse` 模式），UI 显示加载状态。

## 九、使用操作说明

### 9.1 导入课程包

1. 在 APP 首页点击"边学边答"入口
2. 进入学习主页，点击"导入课程"按钮
3. 系统打开文件选择器，选择 `.json` 格式的课程包文件
4. 系统自动解析课程包，成功后课程出现在列表中
5. 若导入失败，会显示具体错误提示（如格式不合法、缺少 course 节点等）

### 9.2 课程包 JSON 格式要求

用户需按第四节定义的格式准备 JSON 文件：
- 必须包含 `course` 节点（含 `courseId`、`courseName`、`sections`）
- 必须包含 `questions` 数组（标准题目格式）
- 题目的 `knowledgePoints` 需与章节的 `id`/`questionTag` 匹配

### 9.3 学习流程

1. 在学习主页点击某门课程 → 进入章节列表
2. 查看总体进度看板，选择要学习的章节
3. 进入学习会话页，阅读知识点讲解（可展开/折叠）
4. 点击"开始练习" → 答题巩固
5. 提交答案后查看正误与解析
6. 完成所有题目后查看练习总结
7. 可选择"重新练习"或"下一章节"

### 9.4 学习进度查看

- **章节列表页**：每个章节显示学习状态（未学/已学/需巩固/已掌握）和正确率
- **课程列表页**：每门课程显示已学章节/总章节、平均正确率
- 进度自动持久化，重启 APP 后保持

### 9.5 错题复习

学习模式中答错的题目自动进入"错题本"，可在错题本中针对性复习，支持间隔复习机制。

## 十、Assumptions & Decisions（假设与决策）

1. **通用课程导入，不绑定固定课程，不生成内置格式数据**：系统通过「课程包 JSON」导入任意教程+题目，用户自行准备课程包文件。
2. **课程包格式**：一个 JSON 文件同时包含 `course`（教程章节）和 `questions`（题目），通过 `section.id` ↔ `question.knowledgePoints` 关联。
3. **独立导入入口**：课程包导入走 StudyScreen 的独立入口，不经过现有 ImportScreen，避免影响现有题库导入功能。
4. **不强制前置依赖**：前置章节未学完时显示提示但不阻止进入下一章节。
5. **题目复用现有题库体系**：课程包的题目作为标准 QuizBank 导入，可通过普通练习模式独立访问。
6. **代码块使用深色背景**：遵循代码编辑器惯例。
7. **知识点讲解用纯文本**：不实现完整 Markdown 渲染器，讲解内容用 `\n` 分段，代码用独立 CodeBlock 展示。
8. **学习进度与课程持久化在 SharedPreferences**：复用现有持久化机制，键 `knowledge_courses`、`study_progress`。
