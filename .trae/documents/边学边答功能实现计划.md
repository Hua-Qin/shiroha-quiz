## 一、功能概述

在 Android 原生端（Kotlin + Jetpack Compose + Material3）实现"边学边答"功能模式。**不绑定任何固定课程，不生成系统内置格式数据**，仅实现通用的「课程包」可导入功能：用户可导入任意"教程 + 配套题目"组成的课程包 JSON 文件，系统自动解析为「知识点章节 + 关联题库」，实现学完每个知识点后立即答题巩固。

### 核心特性

- **通用性**：通过"课程包 JSON"导入任意教程+题目，不绑定特定内容
- **零内置格式数据**：不预置任何课程或题库，全部依赖用户主动导入
- **关联性**：教程章节 ↔ 题目通过 `id` ↔ `knowledgePoints` 双向关联
- **可观测性**：完整的学习进度记录与效果检验
- **可扩展性**：复用现有导入体系（文件选择器、JSON 解析、题库存储）

### 范围

| 范围 | 是否涉及 |
|------|----------|
| 原生 Kotlin 端 | ✓ 主要改动 |
| 现有题库系统 | ✓ 复用 |
| 错题本 | ✓ 复用 |
| 跨端 Web | × 不涉及 |
| 持久化机制 | ✓ 扩展两个 SP 键 |
| 顶层导航 | ✓ 新增三个 MainTab |

## 二、现有导入逻辑架构技术分析

> 本节是对项目现有导入体系的代码级技术分析，为后续扩展"课程包"格式提供架构依据。

### 2.1 整体架构概览

导入系统采用三层架构，数据流清晰单向：

```
UI 层 (ImportScreen.kt)
    ↓ 文本/文件 Uri
解析分发层 (QuizRepository.parseImportJsonPreview / QuizImportParser)
    ↓ ImportResult (题目列表 + 诊断)
数据持久层 (QuizRepository.importBank / appendQuestionsToBank → persist)
```

关键设计特点：**JSON 与纯文本走两条独立解析路径**，在 `ImportScreen.startParse()` 中先尝试 JSON，失败则回退到文本解析器。

### 2.2 UI 层：ImportScreen.kt 文件选择与导入流程

**文件路径**：`apps/android/app/src/native/java/com/yiqiu/shirohaquiz/ui/screens/ImportScreen.kt`

#### 2.2.1 文件选择器实现

使用 Android Storage Access Framework (SAF)，通过 `ActivityResultContracts.OpenDocument()` 注册 launcher：

```kotlin
// 主文件选择器（ImportScreen.kt:255-302）
val filePicker = rememberLauncherForActivityResult(ActivityResultContracts.OpenDocument()) { uri ->
    if (uri == null || isImportBusy) return@...
    selectedFileName = queryFileName(context, uri)
    // 文件大小校验 → 读取字节 → 解码
}
```

启动方式（`ImportScreen.kt:660`）—— MIME 类型使用 `arrayOf("*/*")`，接受所有类型，由解码器根据内容判断格式：

```kotlin
filePicker.launch(arrayOf("*/*"))
```

**关键技术点**：无需任何运行时权限声明。`OpenDocument()` 返回的是临时 `content://` Uri，SAF 自动授予细粒度读取权限。

#### 2.2.2 文件读取链

1. `ContentResolver.query(uri)` 查询文件名（`_display_name` 列）
2. `OpenableColumns.SIZE` 校验文件大小（分警告/阻断两档）
3. `openInputStream(uri)?.readBytes()` 读取全部字节
4. 交给 `QuestionImportAssetExtractor.decodeDetailed()` 解码（支持 docx/txt/json/xlsx/csv + 图片提取）

#### 2.2.3 解析触发与分发（核心：startParse）

`startParse()`（`ImportScreen.kt:348-420`）是解析调度中枢，**分发优先级链**：

1. **JSON 单题库** → 直接构造 `ImportResult`
2. **JSON 多题库** → 抛异常，引导去"数据管理"
3. **双文件文本** → `QuizImportParser.parseDualText()`
4. **标准文本** → `QuizImportParser.parseStandardText()`

```kotlin
fun startParse() {
    importScope.launch {
        val result = runCatching {
            withContext(Dispatchers.Default) {
                // 第一步：优先尝试 JSON 预解析
                val jsonPreview = QuizRepository.parseImportJsonPreview(rawSnapshot)
                // 第二步：多题库 JSON 拦截
                if (jsonPreview != null && jsonPreview.banks.size > 1) {
                    throw IllegalArgumentException("...")
                }
                // 第三步：单题库 JSON 直接构造结果
                val jsonBank = jsonPreview?.banks?.singleOrNull()
                val parsedResult = if (jsonBank != null) { ... }
                else if (dualSnapshot) { QuizImportParser.parseDualText(...) }
                else { QuizImportParser.parseStandardText(...) }
            }
        }
    }
}
```

### 2.3 解析分发层：QuizRepository JSON 格式识别

**文件路径**：`apps/android/app/src/native/java/com/yiqiu/shirohaquiz/state/QuizRepository.kt`

#### 2.3.1 JSON 格式识别核心逻辑

`parseStandaloneJsonPreview()`（`QuizRepository.kt:2373-2422`）是格式分发的关键：

```kotlin
private fun parseStandaloneJsonPreview(rawText: String): JsonImportPreview? {
    val text = rawText.trim()
    return if (text.startsWith("[")) {
        // 分支 A：顶层数组
        val array = JSONArray(text)
        if (looksLikeBankArray(array)) { /* 多题库数组 */ }
        else { /* 题目数组 */ }
    } else {
        // 分支 B：顶层对象
        val root = JSONObject(text)
        val standaloneBank = parseStandaloneBankJsonObject(root)
        when {
            standaloneBank != null -> { /* 单题库 */ }
            root.optJSONArray("banks") != null -> { /* 多题库备份 */ }
            else -> null  // 回退到文本解析
        }
    }
}
```

#### 2.3.2 格式识别决策树

| 输入特征 | 识别 kind | 处理方式 |
|---|---|---|
| `[` 开头 + 首元素含 `questions` | `multi_bank_array` | 多题库，导入页拦截 |
| `[` 开头 + 首元素无 `questions` | `question_array` | 单题库（题目数组） |
| `{` 含 `questions` 字段 | `single_bank` | 单题库 |
| `{` 含 `banks` + Web 标志 | `web_backup` | 多题库 + 跨端状态 |
| `{` 含 `banks` 无 Web 标志 | `multi_bank_backup` | 多题库 |
| 其他 | `null` | 回退到文本解析 |

> **扩展点**：课程包格式（含 `course` 节点）的识别，只需在分支 B 的 `when` 表达式中新增一个分支。

#### 2.3.3 题目解析的高度兼容性

`parseQuestion()`（`QuizRepository.kt:4489-4595`）对所有字段都做了多别名兼容：

- **选项**：支持 `[{key,text}]` 数组、`["选项A"]` 字符串数组、`{"A":"文本"}` 对象三种格式
- **答案**：支持 `answer`/`answers`/`correctAnswer`/`correctAnswers` 四种字段名，支持数组和分隔符字符串
- **题型**：`parseQuestionType()` 支持大量别名（single/single_choice/radio 等）
- **元数据**：已解析 `knowledgePoints`、`difficulty`、`category`、`tags` 等字段

> **关键结论**：题目解析层无需任何改动，课程包的 `questions` 部分可直接复用全部解析能力。

### 2.4 数据持久层

#### 2.4.1 持久化机制

`persist()`（`QuizRepository.kt:4073`）：使用 SharedPreferences，通过单个 `Editor` 链式批量写入后 `.apply()`。

- `PREFS_NAME = "shiroha_quiz_native"`
- **集合类数据**先转 JSON 字符串再 `putString`
- **标量数据**直接 `putBoolean`/`putInt`/`putString`

#### 2.4.2 两种集合序列化范式（扩展参考）

**范式 A：简单 Map**（参考 `practiceSequentialProgress`）—— 适合 `studyProgress`（学习进度）

**范式 B：嵌套对象列表**（参考 `studyRecords`）—— 适合 `knowledgeCourses`（课程列表），展示如何序列化包含嵌套子列表的复杂对象

#### 2.4.3 KEY 常量命名规范

```kotlin
private const val KEY_BANKS = "banks"                          // 全小写 snake_case
private const val KEY_STUDY_RECORDS = "study_records"
private const val KEY_PRACTICE_SEQUENTIAL_PROGRESS = "practice_sequential_progress"
```

### 2.5 完整数据流总结

```
用户操作
  ├─ 选择文件 → filePicker.launch(*/*) → Uri
  │     → ContentResolver.openInputStream → bytes
  │     → QuestionImportAssetExtractor.decodeDetailed → DecodeResult(text)
  │     → rawText = content.text
  ├─ 粘贴文本 → OutlinedTextField → rawText
  └─ 点击"开始解析" → startParse()
        → Dispatchers.Default 异步
        ├─ JSON 路径: parseImportJsonPreview → parseStandaloneJsonPreview
        │     → 识别 kind → JsonImportPreview(banks)
        │     → 单题库: 直接构造 ImportResult(questions)
        └─ 文本路径: QuizImportParser.parseStandardText / parseDualText
              → 多候选竞争 → ImportResult(questions, warnings)
        ↓
  applyParsedResult() → editableQuestions
  ↓ 预览 UI
  ↓ 确认写入
  ├─ NEW_BANK → importBank → banks += → persist()
  └─ APPEND_TO_BANK → appendQuestionsToBank → persist()
```

## 三、课程包格式定义（核心规范）

### 3.1 课程包 JSON 结构

导入的「课程包」JSON 格式如下（兼容现有题目解析器，教程与题目共存于一个文件）：

```json
{
  "course": {
    "courseId": "kotlin-basic",
    "courseName": "Kotlin 程序设计基础（一）基础语法",
    "description": "Kotlin 入门基础语法，边学边练",
    "sections": [
      {
        "id": "ch1_3",
        "chapterTitle": "第1章 走进新语言",
        "sectionTitle": "1.3 程序代码基本结构",
        "order": 2,
        "summary": "一句话摘要",
        "content": "详细讲解（纯文本，\\n 分段）",
        "codeExample": "示例代码",
        "codeExplanation": "代码要点",
        "difficulty": "easy",
        "prerequisites": ["ch0"],
        "questionTag": "ch1_3"
      }
    ]
  },
  "bankName": "Kotlin基础语法题目",
  "questions": [
    {
      "type": "single",
      "question": "Kotlin 程序的入口函数是？",
      "options": [
        {"key": "A", "text": "fun main()"},
        {"key": "B", "text": "public static void main"}
      ],
      "answer": ["A"],
      "analysis": "Kotlin 程序入口是 fun main()",
      "knowledgePoints": ["ch1_3"],
      "category": "1.3 程序代码基本结构",
      "difficulty": "easy"
    }
  ]
}
```

### 3.2 字段说明

#### 3.2.1 顶层字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `course` | object | ✓ | 教程内容节点 |
| `bankName` | string | 可选 | 配套题库名（缺省时用「<课程名> 题目」） |
| `questions` | array | ✓ | 配套题目数组 |

#### 3.2.2 course 节点字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `courseId` | string | ✓ | 课程唯一标识（用于重复导入判断） |
| `courseName` | string | ✓ | 课程名称 |
| `description` | string | 可选 | 课程简介 |
| `sections` | array | ✓ | 知识点章节数组 |

#### 3.2.3 section 节点字段

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `id` | string | ✓ | 章节唯一标识，与题目 `knowledgePoints` 关联 |
| `chapterTitle` | string | 可选 | 大章节标题（如「第1章 走进新语言」） |
| `sectionTitle` | string | 可选 | 小节标题（如「1.3 程序代码基本结构」） |
| `order` | int | 可选 | 学习顺序（缺省按数组下标递增） |
| `summary` | string | 可选 | 一句话摘要 |
| `content` | string | 可选 | 详细讲解（\n 分段） |
| `codeExample` | string | 可选 | 示例代码 |
| `codeExplanation` | string | 可选 | 代码要点 |
| `difficulty` | string | 可选 | "easy" / "medium" / "hard" |
| `prerequisites` | array | 可选 | 前置章节 id |
| `questionTag` | string | 可选 | 题目筛选标签（缺省等于 `id`） |

#### 3.2.4 question 节点字段

完全复用现有 `parseQuestion()` 支持的字段集：

- 必填：`question`、`type`、`answer`
- 推荐：`options`、`analysis`、`knowledgePoints`、`difficulty`、`category`
- 可选：`images`、`score`、`subject`、`grade`、`tags`、`source`、`blankAnswers`

### 3.3 关联规则

题目 `knowledgePoints` 中的值（如 `"ch1_3"`）与教程章节 `id` / `questionTag` 匹配，即构成"教程章节 → 教程题目"的对应关系。

**容错**：

- 若题目未标注 `knowledgePoints`，仍可作为普通题库使用，只是不会出现在任何章节的练习中
- 若章节没有匹配题目，章节学习页显示"本章暂无题目"，可标记为已学习后返回

```kotlin
data class KnowledgeSection(
    val id: String,                      // 章节唯一标识
    val chapterTitle: String,            // 章节大标题
    val sectionTitle: String,            // 小节标题
    val order: Int = 0,                  // 学习顺序序号
    val summary: String = "",            // 一句话核心摘要
    val content: String = "",            // 详细讲解（\n 分段）
    val codeExample: String = "",        // 示例代码
    val codeExplanation: String = "",    // 代码要点说明
    val difficulty: String = "easy",     // "easy" / "medium" / "hard"
    val prerequisites: List<String> = emptyList(),
    val questionTag: String = ""         // 与 Question.knowledgePoints 关联
)

data class KnowledgeCourse(
    val courseId: String,
    val courseName: String,
    val description: String = "",
    val sections: List<KnowledgeSection> = emptyList(),
    val linkedBankId: String? = null,    // 关联的题库 id
    val importedAt: Long = 0L
)

data class SectionProgress(
    val studied: Boolean = false,
    val practiced: Boolean = false,
    val correctCount: Int = 0,
    val totalCount: Int = 0,
    val lastStudiedAt: Long = 0L,
    val bestAccuracy: Float = 0f,
    val practiceCount: Int = 0
) {
    val masteryLevel: MasteryLevel get() = when {
        !studied -> MasteryLevel.NOT_STARTED
        !practiced -> MasteryLevel.STUDIED
        bestAccuracy >= 0.8f -> MasteryLevel.MASTERED
        bestAccuracy >= 0.6f -> MasteryLevel.NEED_REVIEW
        else -> MasteryLevel.NEED_RELEARN
    }
}

enum class MasteryLevel(val label: String) {
    NOT_STARTED("未学习"),
    STUDIED("已学习"),
    NEED_RELEARN("需重学"),
    NEED_REVIEW("需巩固"),
    MASTERED("已掌握")
}
```

fun judgeAnswer(question: Question, userAnswer: List<String>): Boolean {
    return judgeQuestionCore(question, userAnswer.map { it.trim() }.filter { it.isNotBlank() })
}

private fun judgeQuestionCore(question: Question, normalizedUserAnswer: List<String>): Boolean {
    return when (question.type) {
        QuestionType.SINGLE,
        QuestionType.MULTIPLE,
        QuestionType.JUDGE -> normalizedUserAnswer.sorted() == question.answer.sorted() && normalizedUserAnswer.isNotEmpty()
        QuestionType.BLANK -> /* 复用 MultiBlankSupport */
        QuestionType.SHORT -> false
    }
}

## 五、关键技术点解析

### 5.1 课程包格式独立识别

现有 `parseImportJsonPreview()` 已具备完善的 JSON 格式识别能力。课程包导入走 **StudyScreen 的独立入口**，不经过 ImportScreen 的 `startParse()`，因此**无需修改现有导入页的格式识别逻辑**，避免影响现有题库导入功能。

Course package 解析的核心方法 `importCoursePackage` 内部独立走 `JSONObject(text).optJSONObject("course")` 校验路径。

### 5.2 题目解析零改动复用

`parseQuestion()` 已支持 `knowledgePoints` 字段的解析（`parseStringArray`），课程包的 `questions` 部分可直接通过 `importBank()` 导入，无需任何改动。

### 5.3 持久化扩展遵循现有范式

`knowledgeCourses` 使用「嵌套对象列表」范式（仿 `studyRecords`），`studyProgress` 使用「简单 Map」范式（仿 `practiceSequentialProgress`），与现有代码风格完全一致。

### 5.4 文件选择器零成本复用

课程导入的文件选择直接复用 `ActivityResultContracts.OpenDocument()`，无需运行时权限，无需修改 Manifest。

### 5.5 路由兼容性

新格式（5 字段）使用 `split(SEPARATOR, limit = 5)`，对老 SharedPreferences 中旧格式（3 字段）数据**降级兼容**：旧数据恢复时新字段为 `null`。

### 5.6 LaunchedEffect 记录进度

`LaunchedEffect(finished)` + `if (finished)` 守门，确保每次进入完成态时记录一次 `recordSectionResult`。`Unit` 作为 key 的版本在"重新练习"场景不会重新触发。

### 5.7 AnnotatedString 安全作用域

代码高亮用 `buildAnnotatedString { withStyle(SpanStyle(color = ...)) { append(...) } }` 而非 `pushStyle/pop`，避免 pop 缺失导致的运行时报错。

## 六、潜在问题处理方案

### 6.1 课程包格式校验失败

**问题**：用户导入的 JSON 不含 `course` 节点或格式不合法。
**处理**：`importCoursePackage` 内部用 `runCatching` + `JSONObject.optJSONObject("course")` 校验，失败时返回明确的中文错误提示（如"未检测到课程包格式：缺少 course 节点"）。

### 6.2 重复导入同一课程

**问题**：用户多次导入同一个课程包。
**处理**：`importCoursePackage` 检查 `courseId` 是否已存在，若存在：

1. 清理 `studyProgress` 中该 courseId 的所有进度
2. 删除旧 course 的关联 `QuizBank`（通过 `deleteBank()`）
3. 从 `knowledgeCourses` 中移除旧 course
4. 创建新题库、添加新课程

### 6.3 章节无关联题目

**问题**：某个章节的 `questionTag` 没有匹配到任何题目的 `knowledgePoints`。
**处理**：章节仍可正常学习知识点，但"开始练习"按钮显示"本章暂无题目"并提供"标记为已学习"按钮。

### 6.4 课程删除的数据一致性

**问题**：删除课程时需同步删除关联题库和进度。
**处理**：`deleteCourse()` 依次清理：

1. `studyProgress` 中该 courseId 的所有进度（按 key 前缀过滤）
2. `knowledgeCourses` 中的课程
3. 关联的 `QuizBank`（调用现有 `deleteBank()` 自动处理错题本/记录）

### 6.5 大文件导入性能

**问题**：课程包文件较大时解析可能卡顿。
**处理**：解析逻辑放在 `Dispatchers.IO` 异步执行（参考 `startParse` 模式），UI 显示加载状态。

### 6.6 中文文本内容的章节 ID 关联

**问题**：如果 `section.id` 含特殊字符（如点、横线），可能与 `knowledgePoints` 不匹配。
**处理**：建议在 JSON 中使用统一的 ASCII id 命名（如 `ch1_3`），避免中文字符。系统对任意字符串不做限制。

### 6.7 关联题库被外部删除

**问题**：用户在"题库管理"中误删了关联题库。
**处理**：`questionsForSection()` 中通过 `bank?.let { banks.firstOrNull { it.id == id } } ?: return emptyList()` 安全处理，无关联题库时返回空列表。

- 删除会同步清理：所有章节进度 + 关联题库 + 题库内的错题/收藏/记录引用

## 九、假设与决策

1. **通用课程导入，不绑定固定课程，不生成内置格式数据**：系统通过「课程包 JSON」导入任意教程+题目，用户自行准备课程包文件。
2. **课程包格式**：一个 JSON 文件同时包含 `course`（教程章节）和 `questions`（题目），通过 `section.id` ↔ `question.knowledgePoints` 关联。
3. **独立导入入口**：课程包导入走 StudyScreen 的独立入口，不经过现有 ImportScreen，避免影响现有题库导入功能。
4. **不强制前置依赖**：前置章节未学完时显示提示但不阻止进入下一章节。
5. **题目复用现有题库体系**：课程包的题目作为标准 QuizBank 导入，可通过普通练习模式独立访问。
6. **代码块使用深色背景**：遵循代码编辑器惯例。
7. **知识点讲解用纯文本**：不实现完整 Markdown 渲染器，讲解内容用 `\n` 分段，代码用独立 CodeBlock 展示。
8. **学习进度与课程持久化在 SharedPreferences**：复用现有持久化机制，键 `knowledge_courses`、`study_progress`。
9. **AnnotationString 高亮使用 withStyle 安全作用域**：避免 pushStyle/pop 不匹配的运行时报错。
10. **判分逻辑抽取公开化**：将原本私有 `evaluateQuestion` 中的核心判分抽出为 `judgeQuestionCore`（private）+ `judgeAnswer`（public），供学习会话页调用。
