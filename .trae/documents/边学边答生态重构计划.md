# 边学边答生态重构计划（默认首页 + 数据更新 + 模块整合）

## Summary

围绕"边学边答"作为唯一答题入口的战略，对项目进行全面重构：
1. **彻底删除独立刷题模式**：移除 `MainTab.Practice`，答题仅通过 `StudySessionScreen`；`PracticeScreen`/ExamScreen 保留作为底层实现但不再有底栏/首页入口
2. **重塑默认首页**：`StatisticsScreen` 接管启动默认 Tab 角色，原 HomeScreen 内容整体迁移到 Statistics 内（成为"今日学习"区）
3. **修复数据同步链路**：StudySessionScreen 答题每题立即写入错题本 + 学习进度 + studyRecord；StatisticsScreen 改为响应式订阅
4. **整合错题本/学习记录/收藏**：三模块的入口在 Statistics 内直接可达，并整合到首页统计卡片里

## Current State Analysis

### 项目现状

**导航层面** (`ShirohaAppShell.kt:97-127`)
- 25 个 MainTab，底栏可见仅 4 个：`Home/Practice/Import/Me`
- 启动默认 tab：`ShirohaAppShell.kt:193` `var currentTab by rememberSaveable { mutableStateOf(MainTab.Home) }`
- 不在底栏的相关 Tab：`WrongBook / Favorites / Records / Statistics / Study / StudyCourse / StudySession`，均靠 HomeScreen 入口跳转

**首页层面** (`HomeScreen.kt:100-184`)
- `ShirohaHeader` → `IllustrationHeroCard` → `TodayStatusCard` → `StudyEntryCard` → `StatisticsEntryCard` → `HomeShortcutGrid`（错题本/收藏/题库管理/学习记录 2x2）

**统计页面层面** (`StatisticsScreen.kt:67`)
- `val data = remember { QuizRepository.computeStudyStatistics() }` — **关键问题**：`remember{}` 无 key，缓存一次后不自动重算；用户答一题后回到 Statistics 不会自动刷新（需离开/重入页面）

**学习模式数据流问题** (`StudySessionScreen.kt:162-167`)
- 当前 `onSubmit` 仅 `correctCount++`，未调用任何持久化函数
- **不调用 `addWrongContext`** → 错题不进入 wrongBook
- **不调用 `markWrongQuestionRight`** → 已有错题的 rightCount 不更新
- **不调用 `recordPracticeResult` 或 `finishPracticeSessionIfNeeded`** → studyRecord 完全不增长
- **仅在 phase=DONE 时** `recordSectionResult` 更新 studyProgress（中途退出则完全丢失）

**统计聚合层面** (`QuizRepository.kt:5344-5387` `computeStudyStatistics`)
- `totalQuestionsAnswered = records.sumOf { it.total }`
- `practiceCount = records.count { it.source == "练习" }` — **严格匹配 "练习"**，不含 StudySessionScreen 数据源
- `examCount = records.count { it.source.contains("考试") }` — 考试独立计算
- `wrongBookSize` 字段已计算但未在页面渲染

**模块入口现状**
- HomeScreen 内 HomeShortcutGrid 已有 4 个快捷入口卡（错题本/收藏/题库管理/学习记录），但 StatisticsScreen 没有任何进入这三个模块的跳板
- Statistics 函数签名仅 `onBack`，缺 `onOpenWrongBook / onOpenRecords / onOpenFavorites / onOpenBankList` 等回调

### 关键发现

1. **StudySessionScreen 中答题数据完全不上报到 wrongBook / studyRecords**：用户主要通过"边学边答"模式答题，但所有数据写入路径（`addWrongContext / markWrongQuestionRight / finishPracticeSessionIfNeeded / recordPracticeResult`）都未被学习模式触发 → StatisticsScreen 永远看不到 Study 模式下答的题。

2. **StatisticsScreen 的 `remember{}` 无 key**：即使上述修好，进入 Statistics 后答一题不会自动重算；解决办法是用 `LaunchedEffect(Unit)` 或 `derivedStateOf` 包裹到响应式 state。

3. **启动默认 Home = ShirohaAppShell.kt:193 那一行**：`mutableStateOf(MainTab.Home)` 是唯一决定启动 tab 的代码。

4. **底栏渲染逻辑在 ShirohaAppShell.kt:298 + 522**：`MainTab.entries.filter { it.showInBottomBar }` 通过枚举属性过滤。新底栏方案需要重新指定 4 个可见的 tab。

5. **StudySessionScreen 的两条 LaunchedEffect** (当前在 `StudySessionScreen.kt:115` 与 `:121-125`)：
   - `LaunchedEffect(sectionId) { markSectionStudied(...) }` — 进入页面就标记已学习
   - `LaunchedEffect(phase)` — 仅 phase=DONE 时 recordSectionResult

### 模块二：数据更新机制修复

#### 2.1 `StudySessionScreen.kt` — 每题立即写入数据

**关键修复**（`StudySessionScreen.kt:162-167` `onSubmit`）：

```kotlin
private fun onSubmit(currentQuestion: Question, selectedAnswer: List<String>) {
    val isCorrect = QuizRepository.judgeAnswer(currentQuestion, selectedAnswer)
    if (isCorrect) {
        correctCount++
        // 立即更新已有错题
        QuizRepository.markWrongQuestionRight(bank, currentQuestion)
        QuizRepository.persist()  // 或复用 addWrongContext 后再 persist
    } else {
        // 答错：立即加入错题本
        val bank = QuizRepository.courseById(courseId)?.linkedBankId?.let { bid ->
            QuizRepository.banks.firstOrNull { it.id == bid }
        }
        if (bank != null) {
            QuizRepository.addWrongContext(
                bank = bank,
                question = currentQuestion,
                userAnswer = selectedAnswer,
                source = "边学边答",
                correctAnswer = currentQuestion.answer,
                addedManually = false
            )
        }
    }
    // 立即写入 studyRecord（逐题）
    QuizRepository.recordPracticeResult(
        questionId = currentQuestion.id,
        bankId = bank?.id,
        bankName = bank?.name ?: "",
        source = "边学边答",
        userAnswer = selectedAnswer,
        correct = isCorrect,
        correctAnswer = currentQuestion.answer
    )
    // studyProgress 更新（章节级）
    val practiceKey = "$courseId::$sectionId"
    val current = QuizRepository.studyProgress[practiceKey] ?: SectionProgress()
    QuizRepository.studyProgress[practiceKey] = current.copy(
        practiced = true,
        correctCount = current.correctCount + (if (isCorrect) 1 else 0),
        totalCount = current.totalCount + 1,
        lastStudiedAt = System.currentTimeMillis(),
        bestAccuracy = ((current.correctCount + (if (isCorrect) 1 else 0)).toFloat() / (current.totalCount + 1)).coerceIn(0f, 1f),
        practiceCount = current.practiceCount + 1
    )
    QuizRepository.persist()
    submitted = true
}
```

**`recordPracticeResult` 方法改造**（`QuizRepository.kt:3903`）：

当前是死代码（无调用方），改造为：
- 接收 `questionId / bankId / bankName / source / userAnswer / correct / correctAnswer` 等参数
- 写入 `studyRecords.add(0, StudyRecord(id = "practice_${now}", source = source, total = 1, correct = if (correct) 1 else 0, questionResults = [StudyQuestionResult(question, userAnswer, correct, correctAnswer.joinToString(" "))], ...))`
- 末尾 `persist()`

这样 StatisticsScreen 的 `practiceCount` / `totalQuestionsAnswered` / `dailyTrend` 都能立即反映学习模式数据。

## Assumptions & Decisions

1. **彻底删除独立刷题模式**：用户已确认 `MainTab.Practice` 不再作为独立入口，仅作内部实现（PracticeScreen 保留供 WrongBookScreen "练习未掌握错题"按钮等使用，但不直接被用户通过 Home/底栏触达）。所有用户主动答题都走 Study 路径。

2. **首页完全替换为 Statistics 语义**：原 HomeScreen 内容整体迁移到 Statistics/新 HomeDashboardScreen 内；`MainTab.Home` 保留为可跳转的占位 Tab（默认重定向到 Statistics）。

3. **StudySessionScreen 答题每题立即上报**：每题一次 `persist()` 写盘，可接受性能开销（Android SP apply 异步、不阻塞 UI）。

4. **practiceCount 字段含"边学边答"**：扩展 `computeStudyStatistics` 内的 `source` 过滤集合为 `{"练习", "边学边答"}`，让统计数字能反映学习模式。

5. **`wrongBookByCategory` 上次已正确**：上次 Task 6 子代理已实现按 category 聚合，本计划沿用。

6. **studyRecord 来源不重写**：保留 `StudyRecord.source` 字段语义。新增 source 值 `"边学边答"`。

7. **响应式刷新方案选择 remember(key)**：相比 `LaunchedEffect`/`produceState` 周期刷新，`remember(key1, key2, ...)` 更轻量，符合 Compose 习惯。

8. **PracticeScreen 不删除**：PracticeScreen.kt 保留供内部使用（错题本练习等），仅删除 MainTab.Practice 这一独立入口。
